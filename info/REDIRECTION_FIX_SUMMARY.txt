## What You Asked For
"When I click on the event, it's being redirected incorrectly...
Can we do it so that it actually redirects properly to the actual initial website page?"

## What Was Fixed

### The Issue
Events were being found and scraped correctly, but when users clicked on them in the website, the redirect logic was problematic. Links weren't going directly to where the events were scraped from.

### The Solution
Replaced the complex JavaScript redirect with direct HTML links that use the exact `source_url` from the database.

## How It Works

### Before (Complex)
```javascript
onclick="openEvent('{{ ev.source_url }}')"
// URL passed through JavaScript
// Could have encoding/parsing issues
// Custom redirect logic
```

### After (Simple & Direct)
```html
<a href="{{ ev.source_url }}" target="_blank" rel="noopener noreferrer">
  <!-- Event card content -->
</a>
// Direct HTML link to the scraped URL
// Browser handles the redirect natively
// No possibility of misrouting
```
